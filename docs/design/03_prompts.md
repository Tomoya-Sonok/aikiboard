## はじめに（Claude Design 画面での準備）

### A. 「Start with context」にアップロードするもの

1. **Design System** ボタン → `02_tokens.css` を追加（最重要：色・タイポ・余白・10道場テーマが全部入り）
2. 「Drop files here」（右キャンバス下）に下記をドラッグ&ドロップ:
   - `01_design-brief.md`（この Brief）
   - `/projects/aikiboard/docs/requirements.md`（要件定義書の正式版）
   - `/projects/aikiboard/docs/draft/mock.tsx`（既存の React モック、レイアウト＆データ参考）
   - `/projects/aikinote/frontend/src/styles/variables.css`（姉妹サービスのトークン、姿勢合わせ）

### B. プロンプト欄のモード

画面下の「Wireframe」チップが付いていたら **× で外す**。フェーズ 1 は高忠実度を作りたいので、デフォルト（hi-fi）のままで良い。

## ①初回プロンプト（そのまま貼り付け）

```
# プロジェクト
AikiBoard — 合気道の道場運営のための管理＆コミュニケーションプラットフォーム。
詳細はアップロードした `01_design-brief.md` と `requirements.md` を第一参照元としてください。

# 今回つくってほしいもの
PC 1440px ベースで、以下 3 画面を **クリックで行き来できるハイファイ・プロトタイプ** として 1 つのプロジェクトに並べてください。

1. 管理者ホーム（ダッシュボード）
2. 稽古カレンダー（2026 年 7 月、月表示）
3. 稽古詳細モーダル（カレンダーの「7 月 3 日」をクリックで開く）

データは `01_design-brief.md` 末尾のサンプルをそのまま使ってください（蕨合気道会、田中 一郎 ほか）。

# デザインシステム
アップロードした `02_tokens.css` を canonical としてください。
- 色は必ず CSS 変数 `--ab-*` を参照（生の hex を書かない）
- 左サイドバーは `--ab-surface-nav`（墨）、メインキャンバスは `--ab-surface-canvas`（和紙）、カードは `--ab-surface-card`（白）
- アクセントは `--ab-theme-primary`。蕨合気道会は `data-dojo-theme="sumi"`、養神館は `data-dojo-theme="fukamidori"` で切り替える想定
- 角丸はカード 8px、モーダル 12px、ボタン 6px（直線的・シャープ）
- フォント: ブランドロゴ＆ページタイトルは Zen Old Mincho、UI は Noto Sans JP + Inter、数値は Inter

# スタイル方向
「管理ツールらしい直線的・シャープさ」を最優先。情報密度は Linear/Notion 寄り。影は控えめ、罫線で整理。和の気配は色（銅のアクセント、和紙の背景）で残す。

# 参考モック
`mock.tsx` の Sidebar / Calendar / EventModal コンポーネントが視覚的な出発点。そのレイアウトの「骨格」は踏襲、「見た目」は本ブリーフとトークンでアップデートしてほしい。

# 出力
3 画面 + SP 版 3 画面（全 6 フレーム）をキャンバスに並べる。最初の 1 ラウンドで骨格を組み立て、その後こちらから個別にインライン修正します。
```

## ②フォロー用プロンプト（画面別・症状別）

貼り付けるときはチャット欄。**インラインコメントで特定要素を指定できる** 場合はそちらが速い（「この日付セル」など）。

### B-1. サイドバーを整える
```
サイドバーの道場アイコンを、正方形 36x36 角丸 10px に統一して、選択状態は左端に 3px の銅（--ab-dou）バー＋背景を --ab-dou に 12% alpha で塗ってください。アイコン下には道場名を 12px で 1 行省略表示。
```

### B-2. カレンダーのセル密度を下げる
```
カレンダーの各日付セルを高さ 96px 以上、上下余白 8px、日付番号は右上、稽古ドット（--ab-dou）は左下に。土日はセル背景を --ab-washi-dark に。今日の日付は二重丸（外側 --ab-sumi、内側 --ab-dou）にしてください。
```

### B-3. 稽古モーダルを「完成形」に近づける
```
モーダルは幅 560px、角丸 12px、ヘッダーに日付と「×」、本文上段は日時／場所／指導者の 3 項目をテーブル風 2 カラムで（ラベル左、値右）、中段は「参加」「不参加」ボタンを大きく横並び（高さ 44px、銅と墨の 2 トーン）、下段は参加者アバター + 氏名のグリッド、下に不参加者を一行で小さく。右上に管理者用の「編集」メニュー。
```

### B-4. ホームの右カラムを埋める
```
ホーム右カラムに「今月の出欠集計」ミニチャートを追加。縦棒 4 本（4 週）、y 軸は参加人数、--ab-dou で塗る。下に「詳細 →」のテキストリンク。さらにその下に「アクティビティログ」を 6 件、タイムスタンプ＋アイコン＋本文の 1 行形式で。
```

### B-5. SP 版の整形
```
SP 版（横 390px）ではサイドバーをハンバーガードロワーに、ホームの 2 カラムを縦積みに、カレンダーは月切り替えヘッダー + セル高さ 64px に、モーダルはフルスクリーンシート（上から 16px、残り全域）に変換してください。
```

### B-6. トーンを調整したい
- もう少し「和」寄りに → `和紙のテクスチャを 3% 程度、サイドバーの墨の表面にごく薄い紋様を足して、全体は質感は残しつつフラットに保ってください`
- もう少し「SaaS」寄りに → `和紙背景を純白に寄せて（#FAFAF8）、銅のアクセントを減らし、墨のサイドバーを #1F2024 にしてよりモダンな管理ツールの見た目に`

### B-7. 道場テーマ切替のバリエーション
```
同じ「管理者ホーム」を、`data-dojo-theme` を "dou"（銅）／"ai"（藍）／"enji"（臙脂） に差し替えた 3 パターン並べて見せてください。テーマカラーの適用箇所は、サイドバー選択バー／主要 CTA ボタン／出欠集計チャートの 3 か所に絞って確認したいです。
```

## ③フェーズ 2 以降に使うプロンプト（まだ貼らない）

以下のスコープに進みたくなったら個別に貼る想定。

### お知らせ一覧＋詳細
```
Claude Design のキャンバスに「お知らせ一覧」と「お知らせ詳細（本文リッチテキスト、既読者一覧、メール通知フラグ表示）」を追加してください。管理者向けには「＋新規お知らせ」CTA、メンバー向けには未読バッジ。データは design-brief の 3 件サンプルを使用。
```

### フィード＋スレッド
```
「フィード」画面を追加。投稿カードは本文／画像／AikiNote 引用カード（銅の細い縦罫 + "from AikiNote" ラベル）／♡／💬。クリックでスレッド詳細に遷移。投稿作成モーダルには「AikiNote にも流す」チェックボックスを目立つ位置に。
```

### アーカイブ（階層ツリー）
```
「アーカイブ」画面を追加。左 280px に階層ツリー（2026 > 演武会 > 春の演武会 のようにネスト）、右にページ本文エディタ的ビュー。管理者のみ「＋子ページ追加」ボタン表示。ツリーのアイコンは Phosphor の FolderOpen / File を踏襲の想定で。
```

### 会計
```
「会計」画面を追加。上段にメンバー月謝ステータス一覧表（支払済み/未払い/未設定のステータスチップ、月選択）、下段に月次収支の縦棒グラフ（--ab-success / --ab-danger）と、支出手入力フォーム。オーナー／アドミンのみ閲覧。
```

### 公開ページ
```
`aiki-board.com/d/warabi-aikido` を想定した公開ページを追加。ヘッダーは道場ロゴ＋道場名、ヒーローは道場写真（プレースホルダ）+ 1 文のキャッチ + 「見学申し込み」CTA、以降に 稽古スケジュール（カレンダー埋め込み、公開分のみ）／指導者紹介／アクセス／問い合わせフォーム の順。SEO タイトル・ディスクリプション領域をキャンバス上に注釈で明示。
```

### オンボーディング（新規ボード作成）
```
新規ユーザーのオンボーディング 4 ステップを並べてください。①サインアップ（Apple/Google/メール） → ②道場マスタから選ぶ or 新規追加（警告モーダル: AikiNote 側にも反映される旨）→ ③ボード基本情報（名称、テーマカラー 10 色プリセットから選択）→ ④メンバー招待リンク生成。進捗は上部ステップバー。
```

## Tips

- **インラインコメント**が早い修正手段: キャンバス上の要素をクリックしてコメント。数 px 単位の調整や文言差し替えはこちらが速い。
- **調整スライダー**が自動で現れる領域（色・余白など）があるので、微調整はまずそこを触る。
- 大きく骨格を変えるときはチャット、細部はスライダー／インラインコメント、の役割分担。
- 画面間のクリック遷移は、初回生成後に「7 月 3 日のセルをクリックしたら稽古詳細モーダルを開く動線にしてください」と追加で頼むと通常通る。
- export は最後にまとめて: 骨格が固まったら HTML export → `/projects/aikiboard/docs/design/export/` に保存 → Claude Code へ handoff bundle を渡す、という流れでつなぐ想定。
