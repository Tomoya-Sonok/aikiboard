# AikiBoard（アイキボード） — 要件定義書

## 1. プロダクト概要

### 1.1 コンセプト

AikiBoard は、合気道の道場・会などのコミュニティを運営する管理者向けの**道場管理＆コミュニケーションプラットフォーム**。個人向け稽古日誌アプリ **AikiNote** の姉妹サービスとして、共通アカウント基盤のもとで連携しながら提供する。

**メタファー**: 道場の掲示板（Board）。AikiNote が個人の「ノート」であるのに対し、AikiBoard は道場に所属するメンバー全員が集まる「掲示板 / ダッシュボード」。

### 1.2 ポジショニング

| | AikiNote | AikiBoard |
|---|---|---|
| 対象 | 個人の合気道家 | 道場・会の運営者＆所属メンバー |
| 主な価値 | 稽古日誌・個人記録 | スケジュール管理・コミュニティ運営 |
| 公開範囲 | 個人所有、任意で共有 | 道場単位のクローズドコミュニティ＋公開ページ |
| 関係性 | 姉妹サービス（共通アカウント） | 姉妹サービス（共通アカウント） |

### 1.3 LINEグループとの差別化

現状、多くの道場がLINEグループで代替しているため、以下の付加価値で差別化する：

- **稽古カレンダー**の視覚的な管理（公開ページとしても機能）
- **出欠管理**（参加表明＋集計）
- **AikiNote連携**による稽古日誌の引用共有
- **道場HP機能**（未ログインユーザーにも公開可能なページ）
- **構造化されたコミュニケーション**（お知らせ / フィード / スレッド）
- フィード投稿をAikiNoteの道場アカウント経由で外部にも流せる連携

## 2. ターゲットユーザー

### 2.1 メインターゲット道場規模

- **小〜中規模**: 5〜40人程度
- 将来的には全規模をカバー

### 2.2 ユーザーロール（3階層）

| ロール | 説明 | 人数 |
|---|---|---|
| **オーナー** | 道場の最上位管理者。ボードの作成者。プレミアムプランの支払い責任者。1ボードに**1名のみ** | 1 |
| **アドミン** | 幹部・事務局メンバー。オーナーと同等の管理操作が可能（オーナー譲渡・ボード削除を除く）。**複数名**登録可 | 複数 |
| **メンバー** | 一般の道場生。閲覧・参加表明・フィード投稿・スレッド参加が可能 | 複数 |

### 2.3 権限マトリクス

| 操作 | オーナー | アドミン | メンバー |
|---|---|---|---|
| ボード設定変更（ロゴ・テーマカラー等） | ✅ | ✅ | ❌ |
| 稽古スケジュール作成・編集・削除 | ✅ | ✅ | ❌ |
| 稽古への参加/不参加表明 | ✅ | ✅ | ✅ |
| お知らせ作成・配信 | ✅ | ✅ | ❌ |
| フィード投稿 | ✅ | ✅ | ✅ |
| スレッド返信 | ✅ | ✅ | ✅ |
| メンバー招待 | ✅ | ✅ | ❌ |
| メンバー削除・停止 | ✅ | ✅ | ❌ |
| アドミン任命・解除 | ✅ | ❌ | ❌ |
| オーナー譲渡 | ✅ | ❌ | ❌ |
| ボード削除 | ✅ | ❌ | ❌ |
| プレミアムプラン管理・支払い | ✅ | ❌ | ❌ |
| アクティビティログ閲覧 | ✅ | ✅ | ❌ |
| メンバー一覧・プロフィール閲覧 | ✅ | ✅ | ✅ |
| 自主退会 | — | ✅ | ✅ |

## 3. 機能要件

### 3.1 稽古カレンダー（最優先）

#### 3.1.1 カレンダー表示
- **月表示固定**（1ヶ月ごとに前月・翌月へ切り替え）
- 稽古がある日付にはドットやバッジ等のインジケーターを表示
- 日付タップで**モーダル表示**: 日時、場所、担当指導者（任意）、備考、出欠状況
- 定期稽古（繰り返しイベント）の設定対応

#### 3.1.2 公開設定
- 管理者がボード単位で**公開 / 非公開**フラグを設定可能
- 公開時: 未ログインユーザーもカレンダーを閲覧可能（道場の公開ページとして機能）
- 非公開時: ログイン済みメンバーのみ閲覧可能
- 想定では95%以上の道場が「公開」を選択

#### 3.1.3 出欠管理
- メンバーが各稽古イベントに対して**参加 / 不参加**を表明
- ログイン済みユーザーには、カレンダーの稽古イベントモーダル上で**参加/不参加を表明したメンバーのプロフィール画像・氏名がひと目でわかる**ように表示
- 出欠状況の集計ビュー（**管理者のみ**閲覧可能なダッシュボード）

### 3.2 お知らせ・連絡事項

- 管理者（オーナー・アドミン）→ メンバーへの**一方向配信**
- メンバー全員への**通知付き**（通知手段は3.7参照）
- お知らせ一覧画面（既読 / 未読管理）
- リッチテキスト対応（見出し・太字・リンク程度）

### 3.3 道場内フィード

- メンバー・管理者の**双方**が自由に投稿可能
- AikiNote の「みんなで」機能の投稿一覧フィードと同じイメージ
- 投稿形式:
  - テキスト
  - 画像・動画
  - **AikiNote 稽古日誌の引用共有**（AikiNote連携）
- 各投稿に対して**スレッド形式**で返信・議論可能（フラット1階層）

#### 3.3.1 AikiNote連携（フィード関連）
- AikiBoard での投稿・スレッドを、AikiNote の道場アカウント経由で AikiNote 側のフィードにも流せる
- AikiNote 側に外部向けAPIを新設して実現
- AikiNote の稽古日誌をAikiBoard フィードに引用共有可能

### 3.4 メンバー管理

#### 3.4.1 メンバー一覧
- 全メンバーのリスト表示（ロール・参加日等）
- 各メンバーのプロフィール閲覧

#### 3.4.2 参加フロー
- **招待リンク**（URL / QRコード）経由での参加
- **AikiNote上の道場データからの参加申請**（AikiNoteの道場マスタと紐付け）
- 参加時の新規登録情報は AikiNote のユーザー作成フローに寄せる
  - AikiBoard のみ利用する場合でも AikiNote のユーザーが自動作成される
  - → 姉妹サービス間のシームレスな行き来を実現

#### 3.4.3 退会・削除
- **メンバー自主退会**: メンバーが自分で道場から退会可能
- **管理者による削除・停止**: オーナー・アドミンがメンバーを削除/停止可能

### 3.5 アクティビティログ

- 管理者向けに、ボード内の操作履歴を記録・閲覧
- 例: 「○○さんが稽古スケジュールを追加」「○○さんがお知らせを投稿」「○○さんが参加表明」
- 管理者（オーナー・アドミン）のみ閲覧可能

### 3.6 道場ページ（公開プロフィール）

#### 3.6.1 MVP
- ロゴ、テーマカラーの設定
- 道場名・基本情報の表示
- 公開カレンダーの埋め込み

#### 3.6.2 将来展望
- 道場HPを持たない小規模道場向けに、**道場ホームページ兼会員向けページ**としてのAikiBoard運用を訴求
- カスタマイズ可能な公開ページ（道場紹介・アクセス・指導者紹介等）

### 3.7 通知（実装優先度: 最後）

- **プッシュ通知**: ネイティブアプリおよびPWA向け
- **メール通知**: 管理者がお知らせ作成時に「メールでも通知する」フラグをONにした場合、メンバーの登録メールアドレスにもメール送信
- 通知対象:
  - お知らせの新規配信
  - 稽古スケジュールの変更・追加
  - （将来）フィード投稿やスレッド返信への通知

### 3.8 複数道場所属

- 1ユーザーが**複数の道場（ボード）に所属可能**
- Slackのワークスペース切り替えUIのような切り替えインターフェース
- 各ボードごとに独立したロール・権限
- 通知は所属ボードごとに管理

## 4. AikiNote 連携仕様

### 4.1 共通アカウント基盤

- AikiBoard と AikiNote は**別サービス**だが、**共通のアカウント（ユーザーテーブル）** を共有
- AikiBoard 新規登録時に AikiNote のユーザーも自動作成
- 認証方式は AikiNote と統一（Apple / Google ソーシャルログイン優先、メール/パスワードはセカンダリ）

### 4.2 道場マスタ紐付け

- AikiBoard のボード作成時に、AikiNote の道場マスタデータ（既存2,223件のAikikai道場 + 将来追加のYoshinkan/USAF道場等）と紐付け可能
- 紐付けにより AikiNote 上から道場への参加申請フローが実現

### 4.3 フィード連携API

- AikiNote 側に外部向けAPIを新設
- AikiBoard のフィード投稿を AikiNote の道場アカウントのフィードにクロスポスト
- AikiNote の稽古日誌をAikiBoardフィードに引用共有

## 8. デザイン方針

### 8.1 方向性

- AikiNote のデザイントークンを**ベース**にしつつ、AikiBoard独自のアレンジを加える
- 姉妹サービスとしての統一感は保ちながら、管理ツールとしての独自性を表現

### 8.2 AikiNote デザイントークン（参考ベース）

- ブランドカラー: 墨 `#2C2C2C` / 銅 `#C4956A` / 和紙 `#F5F3EF`
- ブランド書体: Zen Old Mincho（ブランドレイヤー）
- 機能UI書体: Noto Sans JP
- アイコン: `@phosphor-icons/react`
- Zen / 円相（ensō）のビジュアル言語

### 8.3 AikiBoard でのアレンジ方針（要検討）

- 管理ツールとしての「しっかり感」を加える方向性
- カラーパレットの拡張（例: ボード別のテーマカラー対応）
- 具体的なアレンジ内容はデザインフェーズで決定
