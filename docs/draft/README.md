# docs/draft/

このディレクトリは、AikiBoard の **検討・議論用の草案置き場** です。

## 重要な前提

> **格納物は実装要件ではなく、検討の叩き台です。**
>
> ここに置かれているドキュメント・モック類は、これから慎重な要件定義を進めるための出発点であり、確定仕様ではありません。要件定義が FIX した内容は、本ディレクトリを離れて正規のドキュメント（例: `docs/aikiboard-product-overview.md`、`docs/specifications/*.md` 等）として配置していきます。

## 収録ファイル

| ファイル | 内容 | ステータス |
|---|---|---|
| `requirements.md` | 仮の要件定義書（コンセプト・権限マトリクス・MVP 機能一覧・AikiNote 連携・収益モデル等） | **叩き台** |
| `mock.tsx` | 仮要件のビジュアル叩き台（React コンポーネントの単一ファイル動作デモ） | **叩き台** |

## 各ファイルの注意事項

### `requirements.md`
- ユーザーから提示された仮要件定義書の内容をそのまま保存しています。
- 「未決事項・今後の検討課題」セクションに記載のある項目はもちろん、それ以外の項目も **確定していません**。
- このファイルを根拠に実装を進めるのではなく、要件定義プロセスでの議論用の出発点として活用してください。

### `mock.tsx`
- React + インライン style で構成された単一ファイルのアイデア視覚化です。
- AikiBoard リポジトリにはまだ TypeScript / React の環境構築がされていないため、IDE で開いた際に未解決の依存（`react`, `useState`, `useEffect` 等）が **赤波線** で表示される場合がありますが、これは想定どおりの状態です。環境構築前のリファレンスとして、そのまま保管しています。
- 単一ファイルとしては動作するため、Vite / Next.js 等の React playground にコピペして手元で動かすことが可能です。
