# docs/draft/

このディレクトリは、AikiBoard の **検討・議論用の草案置き場 / 履歴保存領域** です。

## 重要な前提

> **正式版の要件定義書は [`docs/requirements.md`](../requirements.md) に移行しました。**
>
> 当ディレクトリ内の `requirements.md` は **初期叩き台(ヒアリング前のアイデア記録)** として履歴保存しているだけで、**確定仕様を参照する用途では使用しないでください**。内容は古く、正式版と差異があります。

## 収録ファイル

| ファイル | 内容 | ステータス |
|---|---|---|
| `requirements.md` | 初期叩き台(コンセプト・権限マトリクス・MVP 機能一覧・AikiNote 連携・収益モデル等) | **履歴保存(v0.1)** |
| `mock.tsx` | ビジュアルモック(React コンポーネントの単一ファイル動作デモ) | **デザイン参考資料** |

## 各ファイルの扱い

### `requirements.md`(履歴保存)

- ユーザーから最初に提示された仮要件定義書の内容をそのまま保存しています
- 要件定義ヒアリング(2026-04-15)により、内容は大幅に更新・再構成されました
- **確定仕様は [`docs/requirements.md`](../requirements.md) を参照してください**
- 当ファイルは「要件定義がどこから始まり、どう変化したか」を追えるアーカイブとして残しています

### `mock.tsx`(デザイン参考資料)

- React + インライン style で構成された単一ファイルのアイデア視覚化です
- デザインツール **Pencil** での本格的な画面設計を行う際の参考になります
- AikiBoard リポジトリにはまだ TypeScript / React 環境が構築されていないため、IDE で開いた際に未解決の依存が赤波線で表示される場合がありますが、これは想定どおりです
- 単一ファイルとして、Vite / Next.js 等の React playground にコピペすれば手元で動作確認できます

## 将来的な扱い

- Phase 0(リポジトリ初期化)完了後、`mock.tsx` は Pencil の `.pen` ファイルに置き換わる可能性があります
- そのタイミングで本ディレクトリの中身や役割定義は再検討する予定です
